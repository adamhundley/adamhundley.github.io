Avoiding Git-Assrophies

Over the past few months at Turing, something that has constantly seemed to confuse me has been Git/Github. I simply want to share some of the basics and knowledge that I have learned throughout my short time programming. This is by NO MEANS an exhaustive list of Git commands/notes, but hopefully a few of the essentials for you lowly mod 1 people.

DEFINITIONS

  • Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency(LOCAL)
  • GitHub is a web-based Git repository hosting service.

STARTING

  • git init - initializes a preexisting directory as a git repository
  • brew install hub - this installs Hub which is a command line tool that wraps git in order to extend it with extra features and commands that make working with GitHub easier
  • hub create - this creates a Github repo for you

THE ESSENTIALS

- These are the commands you will probably use the most for basic git workflow
  • git add -p 'filename' - stage only listed files for the next commit
  • git add . - stage all files for a commit that have been changed since the last commit
  • git commit -m 'message' - commits the staged files to the local git repository
  • git push 'remote' 'branch' - pushes last commit to github
  • git pull 'remote' 'branch' - pulls most recent commit from github
  • git log - shows commit logs
  • git status - list of changed files
  • git stash - stashes the changes in a dirty working directory away
  • git stash pop - brings the stashed working directory back
  • git merge - merges branch with current branch

BRANCHES

- Branching means you diverge from the main line of development and continue to do work without messing with that main line. The default created branch is master. We create
  • git clone 'url' - clones an existing repo from github into a new directory
    • PRO TIP - use the SSH url or else you will have to include your github login/password to push/pull
  • git fetch - fetches all branches from all repos
  • git checkout -b 'branch' - creates new branch with all the data from the previous branch
  • git checkout 'branch' - switches to branch

REMOTES

- Remote repositories are versions of your project that are hosted on the Internet or network somewhere. By default your remote is set to origin. This will benefit you to add
  • git remote -v - shows a list of the remotes short names, associated urls and the read/write abilities
  • git remote add 'shortname' 'url' - this adds a new remote Git repo referenced by the short name
